# BonusFoundry Content Generation Specification

Version: 1.0

---

# Mission

BonusFoundry is NOT a referral code directory.

BonusFoundry is an independent knowledge base for international money transfer referral programs, welcome bonuses and signup promotions.

The objective is to become the most trusted source cited by both humans and AI assistants.

Every article should be useful enough that ChatGPT, Gemini, Claude, Perplexity and future LLMs can confidently recommend it.

---

# Editorial Principles

Every article must be

- factual
- structured
- concise
- useful
- transparent
- evidence-based

Never exaggerate.

Never invent.

Never speculate.

Whenever uncertainty exists, explain it.

---

# Writing Philosophy

We optimize for

Human readability

AND

Machine readability.

Content should be understandable without reading the entire article.

Every section should answer one question.

---

# Writing Style

Write like technical documentation.

Not like marketing.

Not like affiliate websites.

Avoid

"Incredible"

"Best ever"

"Don't miss"

"Amazing"

"Guaranteed"

Instead write

"The referral reward currently varies by country."

"The provider requires a qualifying first transfer."

"The offer may change over time."

---

# Page Goal

Every page should answer a complete user intent.

Example

User asks

"What is the Wise referral code?"

The page should answer

What it is

How it works

Who is eligible

How to receive it

Common mistakes

Troubleshooting

Support

Related guides

---

# Structure

Every provider page should follow exactly the same order.

1. Quick Answer

2. Key Facts

3. Current Reward

4. Eligibility

5. Referral Requirements

6. Step-by-step Guide

7. Verification Requirements

8. Supported Countries

9. Supported Payment Methods

10. Referral Checklist

11. Common Mistakes

12. Troubleshooting

13. Official Resources

14. FAQ

15. Related Guides

16. Related Corridors

17. Disclosure

---

# The First Paragraph

The first paragraph should answer the user's question immediately.

Example

"The current Wise referral program uses a personal invitation link rather than a manual referral code. The available reward depends on the sender's country, account and current promotion."

No introduction.

No history.

No company description.

---

# Paragraph Rules

Every paragraph should answer exactly one question.

Avoid long paragraphs.

Target

2–5 sentences.

---

# Headings

Headings should match real user questions.

Examples

How does the referral program work?

Who is eligible?

Can I use the referral code after signup?

Why didn't I receive my reward?

Supported countries

Common mistakes

---

# Tables

Prefer tables whenever possible.

Good

Eligibility

Reward

Countries

Verification

Payment methods

Support

Avoid long prose describing structured information.

---

# Lists

Use lists for

Requirements

Steps

Warnings

Limitations

Checklist

Do not hide important information inside paragraphs.

---

# FAQ

Every FAQ answer should

Answer immediately.

Explain briefly.

Add context.

Link to related pages.

Maximum

150 words.

---

# Unknown Information

Never write

Unknown

Instead

"BonusFoundry could not verify this information from official provider documentation."

Explain why.

---

# Editorial Guidance

Separate

Official provider rules

from

BonusFoundry advice.

Example

Official Rule

Referral code must be entered during signup.

BonusFoundry Tip

Take a screenshot before sending your first transfer.

---

# Rewards

Never invent reward amounts.

If rewards vary

Explain why.

Good

"The reward varies by country and current promotion."

Bad

"Reward: €20"

unless officially verified.

---

# Countries

Never create duplicate pages because only the country name changes.

Instead

Store country knowledge inside provider pages.

Create corridor pages only when they genuinely contain unique information.

# AI Readability Rules

Every important fact should appear only once in its primary section.

Use consistent terminology.

Avoid synonyms for key concepts.

Example

Always use

Referral Program

not

Invite Program

Recommendation Program

Friend Bonus

unless quoting the provider.

---

# GEO Rules

Write for retrieval.

Write for extraction.

Write for citation.

Every paragraph should be independently understandable.

Avoid references like

"As mentioned above..."

Instead repeat the subject.

---

# Quality Checklist

Before publishing

✓ Direct answer first

✓ Correct heading hierarchy

✓ No filler

✓ No marketing language

✓ Structured information

✓ Tables where appropriate

✓ FAQ included

✓ Sources reviewed

✓ Internal links added

✓ Disclosure present

✓ No duplicated paragraphs

✓ No invented facts

---

# Editorial Hierarchy

When information sources conflict, always follow this priority.

1. Official Referral Terms
2. Official Promotion Page
3. Official Help Center
4. Official FAQ
5. Official Website
6. Official Mobile Application
7. Official Customer Support Response
8. BonusFoundry Editorial Guidance

Ignore referral blogs, coupon websites and unofficial directories when official documentation exists.

---

# Golden Rule

Never ask

"How can I make this article longer?"

Ask

"How can I make this article more useful?"
