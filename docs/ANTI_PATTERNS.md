# Anti Patterns

Do not

- Create pages only to display a referral code.
- Duplicate content.
- Repeat the same FAQ everywhere.
- Add a database before it's needed.
- Create components for one-time use.
- Add libraries without clear value.
- Optimize for keywords instead of user intent.

Prefer

- Simplicity
- Reusability
- Static content
- Fast pages
- Clear answers
