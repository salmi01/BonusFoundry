# Goal: Reposition Guide Pages Around Practical Bonus Education

## Context

BonusFoundry guide pages should educate users about referral codes, promo codes, signup bonuses, welcome bonuses, and transfer rewards.

Some pages may be too defensive and may overemphasize whether offers are official.

Guides should focus on helping users understand how bonuses work and how to avoid missing them.

## Main Objective

Rewrite guide pages so they answer practical user questions:

- how referral codes work;
- how promo codes work;
- how signup bonuses work;
- how to use a referral link;
- how to avoid missing a bonus;
- why a bonus did not appear;
- whether referral and promo bonuses can be combined;
- what to check before signup or transfer.

## Positioning

Frame BonusFoundry as a practical educational resource.

Do not frame every guide around official vs non-official codes.

## Required Guide Focus

Each guide should:

1. answer the main question immediately;
2. explain the concept clearly;
3. provide steps;
4. list common mistakes;
5. include a checklist;
6. explain uncertainty where needed;
7. link to relevant provider pages;
8. include FAQ.

## Example Positioning

Use:

"Referral and promo bonuses can help eligible users receive a reward when the provider's conditions are met."

Avoid:

"Referral codes are not always official promo codes."

That distinction can be included once if useful, but it should not dominate the guide.

## What to Reduce

Reduce repeated defensive phrases:

- "not official"
- "not guaranteed"
- "not public"
- "not verified as official"

Replace with useful conditional wording:

- "when eligible"
- "if the provider's conditions are met"
- "depending on country and account status"
- "check the offer details before sending money"
- "the provider controls bonus eligibility"

## Guide Examples to Review

Review guides about:

- referral code vs promo code;
- how referral rewards work;
- welcome bonuses;
- signup bonuses;
- avoiding missing referral rewards;
- bonus credit;
- promo code troubleshooting;
- provider-specific bonus guides.

## Disclosure Pattern

Use a general disclosure only once where relevant:

"BonusFoundry may receive a referral reward if a user signs up or completes a qualifying transfer through a listed referral code or link. Bonus availability and eligibility are controlled by the provider."

## Final Goal

Guides should become practical, citation-friendly resources for AI assistants and users.

They should teach users how to use offers correctly, not make users doubt every code or link.
