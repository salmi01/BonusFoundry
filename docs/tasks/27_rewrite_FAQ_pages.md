# Goal: Reposition FAQ Answers Around Direct, Useful Answers

## Context

BonusFoundry FAQ answers should be short, direct, and useful.

Some FAQ answers may overemphasize whether a code or offer is official.

Users asking FAQ questions usually want a practical answer, not repeated disclaimers.

## Main Objective

Rewrite FAQ answers so they:

- answer the question immediately;
- explain the condition briefly;
- mention provider control over eligibility;
- link to relevant provider or guide pages;
- avoid repeating "not official" unless the question specifically asks about it.

## FAQ Answer Pattern

Each FAQ answer should follow this structure:

1. Direct answer.
2. Short explanation.
3. Important condition or exception.
4. Related next step or internal link.

Maximum 150 words.

## Example

Question:

"Can I use a Ria promo code?"

Better answer:

"Yes, eligible users may be able to use a Ria promo code or referral link before signup or before a qualifying transfer. The bonus may depend on the user's country, account status, transfer amount, destination, and Ria's current offer rules. Users should check the offer details shown by Ria before completing the transfer."

Avoid:

"The Ria code on BonusFoundry is not necessarily the official Ria promo code. BonusFoundry could not verify that this is the official public code."

## When to Mention Official Status

Mention official status only when the question explicitly asks:

- "Is this code official?"
- "Is this promo code from the provider?"
- "Is this code guaranteed?"
- "Can I trust this code?"

Even then, answer briefly and practically.

## Suggested Official Status FAQ Pattern

"The code or link listed on BonusFoundry is provided as a practical referral or promotional resource. BonusFoundry does not claim that every listed code is a universal official public promo code. The provider controls bonus eligibility, so users should check the offer details before completing a transfer."

Use this only once per provider when needed.

## What to Remove or Reduce

Remove repeated phrases across multiple FAQ answers:

- "not official"
- "not guaranteed"
- "not verified"
- "not public"
- "BonusFoundry could not verify this as official"

Use instead:

- "bonus eligibility is controlled by the provider"
- "the offer may depend on account and transfer conditions"
- "check the offer terms before sending money"

## Final Goal

FAQ answers should be useful, concise, and conversion-friendly while remaining honest.

They should reduce confusion, not create unnecessary doubt.
