# Goal 2 — Upgrade All Provider Pages

## Context

The AI Content Design System is now available.

Upgrade every provider page to use the new reusable components.

Do not redesign the site.

Do not create new components.

Reuse the design system built in Goal 1.

---

## Objectives

Upgrade every provider page to improve:

- AI extraction
- readability
- scanability
- authority
- consistency

Reduce repetitive wording.

Focus on useful, direct answers.

---

## Apply These Components

Use where relevant:

- Quick Answer
- Key Takeaways
- Key Facts
- Provider Quick Card
- Eligibility Summary
- Reward Summary
- How It Works
- Where to Enter the Code
- Minimum Transfer
- Common Mistakes
- Troubleshooting
- Mini FAQ
- Best For
- Pros / Cons
- Official Sources
- Last Verified
- Related Resources

---

## Editorial Changes

Rewrite provider pages to:

- answer immediately;
- remove repetitive disclaimers;
- reduce unnecessary "may", "can", "should" when facts are known;
- clearly separate verified facts from BonusFoundry advice.

Do not invent information.

If something cannot be verified, say so once.

---

## Provider Quick Card

Every provider should expose at least:

- Reward
- Eligibility
- Minimum transfer
- Where to enter the code
- Last verified
- Official sources reviewed

---

## FAQ

Each provider should include short answers for:

- Who is eligible?
- How do I use the code?
- When do I receive the reward?
- Why didn't I receive the reward?
- Can existing users participate?

---

## Consistency

All provider pages should follow the same structure.

Only the content should differ.

---

## Constraints

- Do not break existing URLs.
- Do not modify routing.
- Do not change SEO metadata unless necessary.
- Preserve existing structured data.
- Reuse existing components.

---

## Deliverables

1. Provider pages updated.
2. Components used.
3. Any duplicated content removed.
4. Build passes.
5. Lint passes.
