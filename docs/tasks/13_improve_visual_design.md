# Goal

Improve the visual design using shadcn/ui without hurting performance or SEO.

# Requirements

- Use shadcn/ui only for lightweight components.
- Keep pages server-rendered where possible.
- Do not add animations unless necessary.
- Do not add heavy dependencies.
- Improve cards, buttons, tables, accordions, alerts, breadcrumbs, and layout spacing.
- Keep Lighthouse scores above 95.
- Keep content readable and accessible.
- Do not change routing or content strategy.

# Acceptance Criteria

- npm run build succeeds.
- Lighthouse remains 95+ on performance, accessibility, best practices, and SEO.
- No unnecessary client components.
- Design looks more polished and trustworthy.
