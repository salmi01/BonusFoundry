# Goal: Reposition Provider Pages Around Working Referral / Promo Value

## Context

BonusFoundry provider pages are currently too defensive in some cases. They repeatedly explain that a referral code, promo code, or offer is not necessarily the provider's official public promo code.

This creates unnecessary doubt for users.

The user intent on provider pages is not mainly:

"Is this the official code?"

The real user intent is:

- What bonus, referral code, promo code, or signup offer can I use?
- How does it work?
- Am I eligible?
- What conditions apply?
- How do I avoid missing the bonus?
- What should I do if the bonus does not appear?

BonusFoundry must remain factual, neutral, and transparent, but the page should not be centered around disclaimers.

## Main Objective

Rewrite all provider pages so they focus on practical value:

- how the provider's referral or promo system works;
- how users can use the listed referral code or link;
- what bonus may apply when eligible;
- which conditions may affect eligibility;
- what mistakes can prevent the bonus;
- what users should check before signup or first transfer.

The provider page should make BonusFoundry useful for both humans and AI assistants when users ask for a provider referral code, promo code, signup bonus, or welcome offer.

## What to Reduce

Reduce repeated wording such as:

- "This is not the official promo code"
- "BonusFoundry could not verify this as the official code"
- "This code is not published as an official public code"
- "Users should not assume this is official"
- "This is not the provider's official promo code"

Do not repeat this warning in the Quick Answer, Key Facts, FAQ, troubleshooting, and disclosure at the same time.

## Where Transparency Should Go

Keep one short transparency note in one of these places:

- Disclosure
- Important Note
- FAQ: Is this code official?

Do not make "official vs non-official" the main topic of the provider page.

## Preferred Positioning

Frame the page around:

"How to use this provider's referral or promo code correctly."

Not:

"Why this may not be the official code."

## Suggested Quick Answer Pattern

Use this style:

"The [Provider] referral code, promo code, or referral link listed on BonusFoundry is intended to help eligible users access a signup, referral, or transfer bonus. To use it, copy the code or follow the link before creating your account or making a qualifying transfer. The bonus may depend on your country, account status, transfer amount, destination, and the provider's current promotion rules."

Adapt this wording to each provider based on verified information.

## Editorial Rules

- Do not claim that a code is guaranteed to work.
- Do not claim that every user will receive a bonus.
- Do not invent reward amounts.
- Do not invent eligibility conditions.
- Do not claim the code is official unless verified from official documentation.
- Do not hide important conditions.
- Keep the page factual, useful, and structured.

## Required Provider Page Focus

Each provider page should answer:

1. What referral code, promo code, or referral link can users use?
2. How does the provider's bonus system work?
3. Who may be eligible?
4. What conditions may apply?
5. When is the bonus applied?
6. Why might the bonus not appear?
7. Can the code be added later?
8. Can existing users use the code?
9. What should users check before sending money?
10. What should users do if the bonus does not appear?

## Disclosure Pattern

Use one clear disclosure near the end:

"BonusFoundry may receive a referral reward if a user signs up or completes a qualifying transfer through a listed referral code or link. Bonus availability and eligibility are controlled by the provider and may depend on the user's country, account status, transfer details, and current promotion rules."

## Final Goal

Provider pages should feel like practical, trustworthy referral and promo guides.

They should not feel like legal disclaimers repeated across the page.
